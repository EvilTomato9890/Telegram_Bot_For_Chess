# Допущения и инварианты

## 1. Один турнир
- В БД хранится только один активный турнир (`tournaments.id = 1`).
- Команды работают в single-tournament режиме.

## 2. Статусы
- `tournament`: `draft -> registration -> ongoing -> finished`.
- `round`: `generated -> ongoing -> closed`.
- `player`: `active | disqualified`.
- `ticket`: `open | assigned | closed`.

## 3. Роли
- В коде роль организатора названа `ADMIN`.
- Для совместимости с БД wire-value оставлен `organizer`.
- `ADMIN_IDS` в `.env` определяет админов.

## 4. Тай-брейки
Сортировка:
1. `score DESC`
2. `buchholz DESC`
3. `median_buchholz DESC`
4. `sonneborn_berger DESC`
5. `rating DESC`
6. `full_name ASC`

Формулы:
- `Buchholz`: сумма очков соперников.
- `Median Buchholz`: `Buchholz` без min/max при 3+ соперниках.
- `Sonneborn-Berger`: сумма `(очки в партии * итоговые очки соперника)`.

## 5. Результаты игр
Канон:
- `1-0`, `0-1`, `0.5-0.5`, `bye`, `forfeit`.

Нормализация `/report`:
- `White|white|Белые|1-0` -> `1-0`
- `Black|black|Черные|0-1` -> `0-1`
- `Draw|draw|Ничья|0.5-0.5` -> `0.5-0.5`

Допущение:
- `forfeit` интерпретируется как победа белых.

## 6. Швейцарские пары
- Базовая сортировка: `score desc`, `rating desc`, `player_id asc`.
- Повторы запрещены строго.
- Если без повторов собрать пары нельзя: требуется `/confirm_next_round`.
- Третий подряд одинаковый цвет penalize (мягкое ограничение).
- Bye назначается игроку без прошлых bye, если возможно.

## 7. Подготовка турнира
Проверки выполняются на `/prepare_tournament`:
- статус `registration`;
- задано число туров;
- есть столы;
- минимум 2 активных участника;
- участников не больше вместимости `2 * tables`;
- столов достаточно для текущего количества партий.

После `/prepare_tournament`:
- игрокам доступна предварительная информация о паре в `/my_next`;
- `/report` недоступен до старта турнира.

## 8. Первый тур после prepare
- Предпросмотр первого тура строится при `/prepare_tournament`.
- Он хранится в `pending_pairing_payload`.
- На `/start_tournament` материализуется именно этот же набор пар.

## 9. Ограничения на approve
- `/approve_result` может менять результат только игр текущего тура.
- После старта следующего тура редактирование прошлого тура запрещено.

## 10. Undo
- `/undo_last_action` доступен только админу.
- Откат применяет последний snapshot перед изменяющей admin-командой.
- Один вызов = один шаг назад.
- После отката бот сообщает, какое действие отменено.
